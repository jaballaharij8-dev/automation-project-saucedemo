Test Multiple Clicks On Menu Button Clics rapides multiples Tester la robustesse du bouton burger face à des clics successifs FAIL (Expected) Résultat attendu – test non bloquant Ce test est volontairement en échec afin de documenter le comportement réel de l’interface lors d’actions utilisateurs rapides.

L’exception ElementClickInterceptedException apparaît lorsque le menu ouvert recouvre le bouton burger, empêchant les clics successifs.

Ce comportement est lié à la conception UI de l’application et ne bloque pas le parcours utilisateur standard.

Le test est conservé en FAIL afin de :
  • Documenter une limitation ergonomique connue
  • Éviter les faux positifs en automatisation
  • Améliorer la compréhension du comportement de l’UI
